长度单位

  • px：绝对单位，像素
  • em：相对单位，相对于父元素的大小的倍数
  • rem：相对单位，但相对的只是HTML根元素
  • pt：绝对单位，72pt=96px
  • vw：相对单位，相对视口宽度
  • vh：相对单位，相对视口高度
  • vmin：相对单位，相对视口宽度或高度中较小的那一个
  • vmax：相对单位，相对视口宽度或高度中较大的那一个
  • %：相对父元素
  • calc()：计算函数

权重

  • !important 最大
  • 计算权重。进制是256，所以基本不会进位。
    行内样式 1000
    id选择器 100
    class选择器 属性选择器 伪类选择器 10
    元素选择器 伪元素选择器 1
  • 权重相同的后面的覆盖前面的
  • 浏览器默认样式 最小

继承

子元素继承父元素的一些css属性
通常和文字相关的都会被继承，可MDN查。
属性值为inherit为强制继承

属性值的计算过程

1.确定声明值：把没有冲突（只有一个，不用比较权重）的自设值确定下来，还有浏览器默认的标签样式值
2.层叠冲突：比较权重，确定有冲突的值应用哪一个
3.使用继承：将还没有值的属性，如果可以继承父类则继承
4.使用默认值：将剩余全部的属性，添加默认值

两个特殊css取值:

  • inherit：强制继承该属性父元素的值
  • initial：初始值，将该属性设为默认值

文字相关

font-size

元素内部文字的尺寸大小

<p>文字</p>

p{
    font-size: 20px;
}
  • px：绝对单位，像素。简单的理解为文字的高度占多少个像素
  • em：相对单位，相对于父元素的字体大小的倍数。最终都会计算成像素大小
  • rem：相对大小，但相对的只是HTML根元素

每个元素必须有字体大小，如果没有声明，则直接使用父元素的字体大小，如果没有父元素，则使用基准字号（浏览器设置的字体大小）。

  • bold：加粗

  • sans-serif：非衬线字体

  • px px：第一个表示横向上占多少像素，第二个表示纵向上占多少像素

常规流（文档流）布局

  • 块盒独占一行，行盒水平依次排列
  • 水平方向：
    • 每个块盒的总宽度，必须刚好等于其父元素的宽度。
    • 水平方向上的auto为自动吸收剩余空间。
    • 未设置width时，width默认为auto，自动吸收剩余空间，即width占所有剩余空间。
    • 设置width时，剩余空间默认吸收到margin-right。
    • width和margin都设置为auto时，优先吸收给width。
  • 垂直方向：
    • 未设置heigh时，盒子高度默认为auto，高度适应内容，由内容撑开。
    • 垂直方向的margin设为auto时，大小为0
  • 百分比取值：
    所有取值相对父元素的width

横向居中，未定高

设置width，并设置左右margin为auto，左右都吸收剩余空间即左右外边距平分剩余空间

margin合并

两个兄弟结构的盒子在垂直方向上的margin是合并的，两个边距取最大值。

<div class="s2"></div>

margin塌陷

两个父子结构的盒子垂直方向上margin相邻会合并

<div class="s1">
        <div class="s2"></div>
</div>

.s1 {
    width: 500px;
    height: 500px;
    margin: 100px;
    background-color: black;
}
        
.s2 {
    width: 100px;
    height: 100px;
    margin: 50px;
    background-color: blue;
}

解决办法:
1.不让两个margin相邻，即给父盒子设置多设置padding或margin
2.给父盒子设置bfc

块级格式化上下文BFC

创建bfc区域，会隔绝内部和外部的联系，内部渲染不影响到外部
1.创建bfc的元素，自动高度会计算浮动元素（可解决浮动高度坍塌问题，但建议使用清除浮动解决）
2.创建bfc的元素边框盒不会与浮动元素重合，会避开浮动元素（可以做侧边栏，侧边栏左/右浮动定宽，主区域bfc元素宽度自适应并会排在浮动元素右边，空隙调浮动元素外边距）
3.创建bfc的元素，不会和子元素进行外边距的合并（解决margin塌陷和margin合并）

能创建bfc的元素：
1.html根元素
2.浮动元素
3.定位元素
4.有overflow:hidden元素

浮动

常用在文字环绕和横向排列

float

  • left：向左浮动，元素靠上靠左
  • right：向右浮动，元素靠上靠右

特点：

  • 元素浮动后都会变成块盒
  • 浮动盒子的子元素为常规流

盒子尺寸：

  • width为auto时，适应内容宽度（常规流时为吸收剩余空间）
  • height为auto时，适应内容高度（和常规流一致）
  • margin为auto时，都为0（常规流时为吸收剩余空间，优先级没width高）
  • 百分比设置相对于父元素的width(和常规流一致)
  • margin并列不会合并（常规流会合并）

盒子排列：

  • 左浮动靠上靠左
  • 右浮动靠上靠右
  • 常规流块盒在前浮动盒子在后，排列时常规流块盒先排列，浮动盒子避开他再排列
  • 浮动盒子在前常规流块盒在后，排列时浮动盒子先排列，常规流块盒无视浮动盒子排列
  • 行盒排列时，会避开浮动盒子（常用作文字环绕）

如果文字没有在行盒中，浏览器会自动生成一个行盒包裹文字，该行盒叫匿名行盒

高度坍塌

常规流盒子高度适应内容，计算时不考虑浮动盒子

<div class="container">
    <div></div>
    <div></div>
    <div></div>
</div>

.container {
    border: 1px solid;
}
        
.container div {
    float: left;
    width: 200px;
    height: 200px;
    background-color: black;
}

解决办法：清除浮动。
1.写一个常规流盒子在浮动元素的后面，让他是一个空元素仅清除全部浮动，即它排列时会看到前面的浮动元素并排在浮动元素后面，并且会撑开他们的父元素常规盒子。（不建议用因为要写一个空元素）

<div class="container">
    <div class="box"></div>
    <div class="box"></div>
    <div class="box"></div>
    <div class="clearfix"></div>
</div>

.clearfix{
    clear:both;
}

.container {
    border: 1px solid;
}
        
.container .box {
    float: left;
    width: 200px;
    height: 200px;
    background-color: black;
}

2.给父元素加一个伪类选择器，类似在子元素的最后加一个元素，让他清除全部浮动，即它排列时会看到前面的浮动元素并排在浮动元素后面，并且会撑开他们的父元素常规盒子。（推荐用）
建议凡浮动，小心高度坍塌问题。在凡浮动元素的父元素上添加clearfix类名，在css样式中如下写该类名的伪类选择器并设置样式。

<div class="container clearfix">
    <div class="box"></div>
    <div class="box"></div>
    <div class="box"></div>
</div>

.container {
    border: 1px solid;
}

.clearfix::after{
    content:"";
    display:block;
    clear:both;
}
        
.container .box {
    float: left;
    width: 200px;
    height: 200px;
    background-color: black;
}

3.给塌陷的盒子创建bfc（定位，浮动，溢出隐藏）

clear

清楚浮动

  • left：清除左浮动，该元素必须出现在前面所有左浮动盒子的下方
  • right：清除右浮动，该元素必须出现在前面所有右浮动盒子的下方
  • both：清除左右浮动，该元素必须出现在前面所有浮动盒子的下方

定位

position

定位

  • static：不定位，默认
  • relative：相对定位
  • absolute：绝对定位
  • fixed：固定定位

特点：
1.定位元素会脱离文档流，相对定位除外
2.文档流元素摆放/计算高度时，忽略脱离文档流的元素
3.元素定位后都会变成块盒，相对定位除外
4.凡被定位后，一定没有浮动，相对定位除外

相对定位

relative：相对定位不会脱离文档流(width值为auto时撑满剩余空间)，只会让元素在原来位置上进行偏移，并且该盒子偏移不会对其他盒子造成任何影响。

绝对定位

absolute：绝对定位脱离文档流(width值为auto时由内容撑开，margin值为auto时撑满剩余空间)
1.当祖先元素设置了定位（relative absolute fixed），则元素会相对于离自己最近的有定位的祖先元素的padding盒进行定位

2.若祖先元素中没有设置定位的元素，则元素会相对于body进行定位

固定定位

fixed：固定定位脱离文档流(width值为auto时由内容撑开，margin值为auto时撑满剩余空间)，他相对浏览器可视窗口定位

定位居中，定高定宽

绝对定位和固定定位，上下左右定位全为0，margin值为auto

<div></div>

div{
    width:200px;
    height:200px;
    background-color:black;
    position: absolute;
    top:0;
    right:0;
    left:0;
    bottom:0;
    margin:auto;
}

z-index

设置元素的叠加顺序
1.越大越在最上层
2.为负数时，会被如果遇到常规流或浮动元素会被覆盖

auto值总结

常规流

width值为auto时撑满剩余空间
height为auto时由内容撑开
margin值为auto时撑满剩余空间
width强于margin

浮动

width为auto时由内容撑开
height为auto时由内容撑开
margin为auto时，都为0

定位

相对定位：
width值为auto时撑满剩余空间
margin值为auto时撑满剩余空间
width强于margin

绝对定位：
width值为auto时由内容撑开
margin值为auto时撑满剩余空间

固定定位：
width值为auto时由内容撑开
margin值为auto时撑满剩余空间

居中问题总结

行盒/行块盒 水平居中

设置其父元素text-align: center

常规流（非浮动、绝对定位和固定定位）块盒 水平居中

设置其元素定宽，左右margin为auto

绝对定位元素 水平居中

绝对定位元素相对其包含块进行定位
设置其元素定宽，left: 0 right: 0，左右margin为auto

单行文本 垂直居中

设置文本所在元素的高度height和文本行高line-height相同

绝对定位元素 垂直居中

设置其元素定高，top: 0 bottom: 0，上下margin为auto

无效图片的宽高问题

无效图片设置宽高无效，他会变成一个普通行盒，建议把img标签设置为块盒，这样图片无效时也不影响其他样式

writing-mode

文字书写方向

  • vertical-rl：从右向左，从上到下
